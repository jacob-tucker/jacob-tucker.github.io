Description

I was lucky enough to begin my journey with Decentology in September of 2020, and I have been working here ever since then. I am currently serving as the lead Cadence developer, which is the smart contract language for the Flow blockchain.

At the moment, I am primarily working on implementing a Non-Fungible Token Engine that allows developers to quickly get decentralized applications running. This means that developers can start creating dApps with out-of-the-box full-stack code along with already set up blockchain accounts/addresses, hot-loaded and hot-deployed smart contract code, and unique action cards to run pre-written transactions and scripts. The developer has the ability to edit all of this in any way they see fit to achieve their goal of a perfect dApp!

More specifically, my primary objective is leading the team in writing the smart contracts, transactions, and scripts in Cadence so the developer has pre-written code, tying these interactions with the blockchain to a library that the developer can use to run those transactions and scripts, and implementing unique action cards that the developer can use to run those library functions with the click of a button on the front-end.
